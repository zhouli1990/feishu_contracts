# 查看合同详情

根据合同 ID 查看合同信息。

## 请求

基本 | 
---|---
HTTP URL | https://open.feishu.cn/open-apis/contract/v1/contracts/:contract_id
HTTP Method | GET
接口频率限制 | [1000 次/分钟、50 次/秒](频控策略 - 服务端 API - 开发文档 - 飞书开放平台)
支持的应用类型 | Custom App
权限要求<br>**调用该 API 所需的权限。开启其中任意一项权限即可调用** | 获取合同信息(contract:contract:readonly)
字段权限要求 | **注意事项**：该接口返回体中存在下列敏感字段，仅当开启对应的权限后才会返回；如果无需获取这些字段，则不建议申请<br>获取用户 user ID(contact:user.employee_id:readonly)

### 请求头

名称 | 类型 | 必填 | 描述
---|---|---|---
Authorization | string | 是 | `tenant_access_token`<br>**值格式**："Bearer `access_token`"<br>**示例值**："Bearer t-7f1bcd13fc57d46bac21793a18e560"<br>[了解更多：如何选择与获取 access token](如何选择使用不同类型的 Token - 开发指南 - 开发文档 - 飞书开放平台)

### 路径参数

名称 | 类型 | 描述
---|---|---
contract_id | string | 合同id<br>**示例值**："6951358043686715692"

### 查询参数

名称 | 类型 | 必填 | 描述
---|---|---|---
user_id_type | string | 否 | 用户 ID 类型<br>**示例值**：open_id<br>**可选值有**：<br>- open_id：标识一个用户在某个应用中的身份。同一个用户在不同应用中的 Open ID 不同。[了解更多：如何获取 Open ID](如何获取用户的 Open ID - 开发指南 - 开发文档 - 飞书开放平台)<br>- union_id：标识一个用户在某个应用开发商下的身份。同一用户在同一开发商下的应用中的 Union ID 是相同的，在不同开发商下的应用中的 Union ID 是不同的。通过 Union ID，应用开发商可以把同个用户在多个应用中的身份关联起来。[了解更多：如何获取 Union ID？](如何获取用户的 Union ID - 开发指南 - 开发文档 - 飞书开放平台)<br>- user_id：标识一个用户在某个租户内的身份。同一个用户在租户 A 和租户 B 内的 User ID 是不同的。在同一个租户内，一个用户的 User ID 在所有应用（包括商店应用）中都保持一致。User ID 主要用于在不同的应用间打通用户数据。[了解更多：如何获取 User ID？](如何获取用户的 User ID - 开发指南 - 开发文档 - 飞书开放平台)<br>**默认值**：`open_id`<br>**当值为 `user_id`，字段权限要求**：<br>获取用户 user ID(contact:user.employee_id:readonly)
info_change_version | int | 否 | 信息修改版本号<br>**示例值**：1
